Specify the Roundtable workspaces if you'd like to track a specific workspace in a repository. If left blank, all workspaces will be examined for changes and built.

Possible options:

  • <workspaceName>
    Tracks/checks out the specified workspace from the remote repository.
    E.g. master, feature1, ...
  • <workspaceName>@<releaseName>
    Tracks/checks out the specified release of the workspace.
    E.g. master@ver-2.3.0
  • ${ENV_VARIABLE}
    It is also possible to use environment variables. In this case the variables are evaluated and the result is used as described above.
    E.g. ${TREEISH}, ...
  • <Wildcards>
    When used the '*' is recognized as a wildcard. Therefore, work* would match work-foo.
